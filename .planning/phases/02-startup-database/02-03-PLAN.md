<task id="3.1">
## Task: Create startup API hooks

Create `frontend/src/lib/startups.ts`:
- `useStartups(filters)` — GET /api/startups with filters, paginated via TanStack Query infinite query
- `useStartup(id)` — GET /api/startups/{id}
- `useSearchStartups(query)` — GET /api/startups/search?q=
- `useSubmitStartup()` — POST /api/startups/submit mutation
</task>

<task id="3.2">
## Task: Build explore page

Create `frontend/src/app/(dashboard)/explore/page.tsx`:
- Header: "Explore Startups" with startup count
- Search bar at top (debounced, 300ms), searches by name
- Filter row: sector dropdown, stage dropdown, score range slider
- Startup list as table rows or card list:
  - Each row shows: name, sector pill, stage badge, momentum score (colored 0-100), domain link
  - "Signals pending" badge for user-submitted startups awaiting signals
- Infinite scroll pagination (load more on scroll)
- Empty state for no results
- Clicking a startup navigates to `/startups/{id}`

Dark theme matching dashboard design.
</task>

<task id="3.3">
## Task: Build startup detail page

Create `frontend/src/app/(dashboard)/startups/[id]/page.tsx`:
- Startup name as header with sector pill and stage badge
- Profile section: domain (clickable), description, founded year, team size
- Momentum score display (large number, colored)
- "Signals pending" notice if signals_status = "pending"
- Placeholder sections for signal charts (Phase 5 will fill these)
- Back button to explore page

This is the detail page shell — signal charts come in Phase 5.
</task>

<task id="3.4">
## Task: Build startup submission flow

Create submit flow (either modal or inline on explore page):
- "Submit Startup" button in explore page header
- Modal/dialog with:
  - URL input field with validation
  - "Submit" button
  - Loading state while enriching
  - Success state: shows extracted startup info
  - Error states: duplicate (shows existing), limit reached (shows upgrade CTA)
- After submission, startup appears in explore list immediately
</task>
