<task id="2.2">
## Task: Build URL enrichment service

Create `backend/app/startups/enrichment.py`:

`async def enrich_from_url(url: str) -> dict`:
- Fetch the URL with `httpx` (timeout 10s, follow redirects)
- Parse HTML with built-in or lightweight parser
- Extract: `<title>`, `<meta name="description">`, `<meta property="og:title">`, `<meta property="og:description">`, `<meta property="og:image">`
- Extract clean domain from URL
- Construct LinkedIn company URL: `https://linkedin.com/company/{domain-without-tld}`
- Return: `{name, description, domain, logo_url, linkedin_url}`
- Handle errors gracefully — return partial data on failure

Install `beautifulsoup4` for HTML parsing.
</task>

<task id="2.3">
## Task: Build startup submission endpoint

Add to `backend/app/startups/routes.py`:

**POST /api/startups/submit**
- Body: `{url: string}`
- Auth required
- Flow:
  1. Extract domain from URL
  2. Check dedup (get_startup_by_domain)
  3. If exists → return 409 with existing startup
  4. Check submission limit (free tier = 10)
  5. If over limit → return 403 with upgrade message
  6. Enrich from URL (extract metadata)
  7. Create startup with source=user_submitted, signals_status=pending, submitted_by=user.id
  8. Return created startup

Add to schemas:
- `StartupSubmitRequest` — url: string
- `StartupSubmitResponse` — startup data + message

Add to services:
- `count_user_submissions(db, user_id)` — count startups where submitted_by = user_id
- `submit_startup(db, url, user)` — orchestrates the full flow
</task>
