## Task: Build startup schemas and service layer

Create `backend/app/startups/schemas.py`:
- `StartupResponse` — id, name, domain, sector, stage, founded_year, description, team_size, momentum_score, score_updated_at, created_at
- `StartupListResponse` — items (list), total, cursor (for pagination)
- `StartupFilters` — sector (optional), stage (optional), min_score (optional), max_score (optional)

Create `backend/app/startups/services.py`:
- `get_startups(db, filters, cursor, limit)` — paginated list with filters
- `get_startup_by_id(db, startup_id)` — single lookup
- `search_startups(db, query, limit)` — name search using `ilike`
- `get_startup_by_domain(db, domain)` — for dedup checking
- `create_startup(db, data)` — insert new startup
- `bulk_create_startups(db, startups_data)` — batch insert for seeding
</task>

<task id="1.2">
## Task: Build startup API endpoints

Create `backend/app/startups/routes.py`:

**GET /api/startups**
- Query params: sector, stage, min_score, max_score, cursor, limit (default 50)
- Returns paginated startup list sorted by momentum_score desc
- Requires auth (get_current_user dependency)

**GET /api/startups/{startup_id}**
- Returns single startup with full profile
- Requires auth

**GET /api/startups/search**
- Query param: q (search term)
- Returns top 20 matches by name (ilike %q%)
- Requires auth

Register router in `backend/app/main.py`.
</task>

<task id="1.3">
## Task: Create seeding infrastructure

Create `backend/scripts/seed_startups.py`:
- Reads from `backend/data/yc_startups.json` and `backend/data/curated_startups.csv`
- Deduplicates by domain
- Assigns sectors from a mapping (YC batch tags → our 10 sectors)
- Assigns stages based on available funding info
- Bulk inserts into database via SQLAlchemy
- Reports: total imported, duplicates skipped, sector distribution

Create sample data files:
- `backend/data/yc_startups.json` — sample structure with 50 example YC startups for testing
- `backend/data/curated_startups.csv` — sample CSV with 20 curated startups

The seeding script should be runnable as:
```bash
cd backend && python -m scripts.seed_startups
```

NOTE: Full 7K dataset will be populated by running actual YC/PH scraping scripts (Phase 3 scraping infrastructure). For now, include a representative sample (~50-100) and the import infrastructure.
</task>
