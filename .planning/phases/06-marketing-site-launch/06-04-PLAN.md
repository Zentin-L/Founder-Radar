<task id="4.1">
## Task: Implement capability detection and tiering

Detection inputs:
- WebGL availability and context creation success
- device memory/core hints + reduced motion preference
- runtime FPS sampling in first seconds

Tier policy:
- Tier A: Full 3D desktop experience
- Tier B: Lite 3D mobile/older laptops
- Tier C: Static image/video fallback for no-WebGL or weak devices
</task>

<task id="4.2">
## Task: Safari and mobile-specific safeguards

WebKit strategy:
- avoid high-cost postprocessing and heavy transparent overlaps
- lower DPR cap and particle counts for Safari/iOS
- texture format fallback to WebP/PNG where compressed formats fail

Gyroscope decision:
- disabled by default for launch
- optional guarded experiment behind feature flag only
</task>

<task id="4.3">
## Task: Accessibility implementation baseline

Include:

- keyboard reachable CTAs and form controls
- ARIA labels and section landmarks
- screen-reader text summaries for major 3D visual states
- prefers-reduced-motion overrides for Lenis and camera animations
</task>
