<task id="1.1">
## Task: Define scene architecture and ownership boundaries

Create a deterministic scene contract:
- `SceneRoot`: canvas, post-processing gates, frame loop mode
- `LightingRig`: ambient + key directional + soft rim point light
- `CameraRig`: controlled by scroll timeline state
- section scene groups mounted/unmounted by visibility windows

Decision answers:
- Startup card primitives: rounded thin boxes (box geometry + beveled shader look)
- Camera animation approach: scroll-driven state + `useFrame` interpolation (no GSAP dependency)
- Lighting: ambient + one directional key + one subtle rim point for premium contrast
</task>

<task id="1.2">
## Task: Scroll integration architecture (Lenis → R3F)

Implement a single source of truth for scroll progress:
- Lenis manages smooth page scroll
- Framer Motion derives section progress values
- R3F camera consumes normalized progress via shared store/context

Velocity mapping:
- clamp scroll velocity to camera acceleration envelope
- map high velocity to eased camera catch-up, not linear jumps
</task>

<task id="1.3">
## Task: Establish rendering budgets and lazy loading

Set initial budgets and guards:
- max visible triangles: ~120k desktop, ~45k mobile-lite
- particle cap: 600 desktop, 180 mobile-lite
- draw calls target: <120 desktop hero, <70 mobile-lite
- texture budget: compressed, max 2K hero, 1K mobile

Lazy-loading strategy:
- defer heavy 3D sections until user reaches hero viewport
- split marketing 3D components into dynamic imports with suspense fallbacks
</task>
