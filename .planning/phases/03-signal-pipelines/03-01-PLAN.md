<task id="1.1">
## Task: Set up Celery worker and beat scheduler

Create worker infrastructure in `backend/app/workers`:
- `celery_app.py` with Redis broker/backend from env
- `schedules.py` registering 12-hour jobs (UTC cron)
- Task routing queues: `jobs`, `linkedin`, `maintenance`

Update `docker-compose.yml` with:
- `celery-worker` service
- `celery-beat` service
- shared env and code volume

Configure task defaults:
- `acks_late=true`
- retries with exponential backoff
- hard/soft time limits
</task>

<task id="1.2">
## Task: Build signal ingestion service foundation

Create in `backend/app/signals/services.py`:
- `record_signal(startup_id, signal_type, value, collected_at)`
- auto-calculation of `previous_value` and `delta`
- idempotency guard for duplicate snapshots in same window
- `get_latest_signal(startup_id, signal_type)` helper

Create schemas in `backend/app/signals/schemas.py` for API/task contracts.
</task>

<task id="1.3">
## Task: Add health monitoring primitives

Create `backend/app/signals/health.py`:
- task run status tracker (success/failure/duration)
- aggregate health snapshot per pipeline
- error counters and last successful run timestamp

Expose read-only endpoint in `backend/app/signals/routes.py`:
- `GET /api/signals/health` (auth required, admin-safe payload)

Include router in `backend/app/main.py`.
</task>
