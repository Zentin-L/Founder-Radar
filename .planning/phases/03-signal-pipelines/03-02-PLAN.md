<task id="2.1">
## Task: Implement job posting collection pipeline

Create `backend/app/signals/jobs_scraper.py`:
- startup domain/job-page discovery strategy (`/careers`, `/jobs`, ATS providers)
- HTTP fetch with polite limits and error handling
- browser fallback for JS-heavy job pages when HTTP parsing fails
- extract posting count and source metadata

Create `backend/app/signals/jobs_parser.py`:
- normalize counts from multiple source formats
- convert to canonical payload `{startup_id, count, collected_at}`
</task>

<task id="2.2">
## Task: Wire Celery task for 12-hour collection

Add task in `backend/app/signals/tasks.py`:
- `collect_job_signals_12h()` (or equivalent 12-hour schedule binding)
- iterates tracked startups in batches
- stores counts via `record_signal(... signal_type=HIRING ...)`
- writes pipeline health telemetry on completion/failure

Use concurrency limits to avoid rate-limit bursts.
</task>

<task id="2.3">
## Task: Compute and expose hiring velocity

In `backend/app/signals/services.py` add:
- `calculate_hiring_velocity(startup_id, window_days=30)`
- uses latest vs prior snapshots and elapsed time
- returns velocity, absolute delta, and trend direction

Add endpoint in `backend/app/signals/routes.py`:
- `GET /api/signals/hiring/{startup_id}` returning time-series and computed velocity
</task>
