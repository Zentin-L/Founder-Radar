<task id="1.1">
## Task: Build hybrid normalized scoring engine

Create `backend/app/scoring/engine.py`:
- `calculate_momentum_score(hiring_velocity, linkedin_growth)`
- normalize each signal to bounded sub-scores before weighting
- apply weighted blend (configurable constants)
- clamp and round final score to `0..100`

Default strategy:
- normalized hiring component
- normalized LinkedIn growth component
- weighted blend suitable for standard depth (transparent and tunable)
</task>

<task id="1.2">
## Task: Persist score and change direction

Update startup scoring fields via service layer:
- write latest `momentum_score`
- write `score_updated_at`
- compute `score_change = new_score - previous_score`
- compute `score_direction` as `up/down/flat`

Ensure startup APIs return these fields for downstream UI use.
</task>

<task id="1.3">
## Task: Add score history table

Add migration + model for historical snapshots:
- `startup_score_history` table with `startup_id`, `score`, `score_change`, `score_direction`, `calculated_at`
- indexed by `startup_id` and `calculated_at`

Store one row per recompute cycle.
</task>
